# OpenROAD Backend Flow ‚Äî Rules & Conventions

This document defines the rules, naming conventions, and process standards
for the sky130hd digital backend flow using Claude Code.

---

## 1. RTL Naming Conventions

| Element | Convention | Example |
|---------|-----------|---------|
| Top module | `lowercase, no underscore suffix` | `alu`, `gcd`, `ibex` |
| Clock port | `clock` (singular, lowercase) | `input clock` |
| Reset port | `reset` (active-high, lowercase) | `input reset` |
| Verilog files | `*.v` | `alu.v` |
| RTL directory | `design/rtl/` | `design/rtl/alu.v` |

**Rule 1.1** ‚Äî Top module name must match `config.mk` `DESIGN_NAME`.
**Rule 1.2** ‚Äî Clock and reset must be declared as `input` ports in the top module.
**Rule 1.3** ‚Äî All memories must be instantiated from sky130hd SRAM macros (not behavioral arrays).

---

## 2. SDC Constraint Rules

**Rule 2.1** ‚Äî Every design must define exactly one primary clock via `create_clock`.
**Rule 2.2** ‚Äî The clock port in `create_clock` must match the port name in the RTL.
**Rule 2.3** ‚Äî Input delays must be set for all inputs: `set_input_delay -clock <clk> 1.0 [all_inputs]`.
**Rule 2.4** ‚Äî Output delays must be set: `set_output_delay -clock <clk> 1.0 [all_outputs]`.
**Rule 2.5** ‚Äî `set_clock_uncertainty` must be specified (minimum 0.3 ns for sky130hd).

---

## 3. Flow Stage Dependencies

```
check ‚Üí generate_config ‚Üí synth ‚Üí floorplan ‚Üí place ‚Üí cts ‚Üí route ‚Üí signoff
```

**Rule 3.1** ‚Äî Never skip stages. The flow must run in dependency order.
**Rule 3.2** ‚Äî If `synth` fails, do not proceed to `place`, `cts`, or `route`.
**Rule 3.3** ‚Äî A stage's output DEF/netlist is the input to the next stage.
**Rule 3.4** ‚Äî `floorplan` must complete before `place`.
**Rule 3.5** ‚Äî `place` (detail placement) must complete before `cts`.
**Rule 3.6** ‚Äî `cts` must complete before `route`.

---

## 4. Tool Invocation Rules

**Rule 4.1** ‚Äî All OpenROAD/Yosys invocations MUST go through the MCP `orfs` server tools.
**Rule 4.2** ‚Äî Never call `docker exec` directly from Claude Code for EDA tools; use `mcp__orfs__*` tools instead.
**Rule 4.3** ‚Äî All Tcl scripts must live under `design/scripts/` and be named `<stage>.tcl`.
**Rule 4.4** ‚Äî All logs must be written to `logs/<stage>.log` inside the container (mirrored to host `logs/`).
**Rule 4.5** ‚Äî Report files must be written to `reports/` with standard names:
  - `signoff_summary.rpt` ‚Äî STA
  - `drc.rpt` ‚Äî DRC
  - `power.rpt` ‚Äî Power
  - `metrics.rpt` ‚Äî Area & cell counts

## 6. File Ownership

| File | Location | Owned by |
|------|----------|---------|
| RTL sources | `design/rtl/` | User (hw1 output) |
| SDC constraints | `design/constraint.sdc` | Generated by Agent |
| Config | `design/config.mk` | Generated by Agent |
| Tcl scripts | `design/scripts/` | Generated by Agent |
| Logs | `logs/` | Written by tools |
| Reports | `reports/` | Written by tools |
| Results | `results/` | Written by tools |

---

## 7. Prohibited Actions

- **Do not** modify sky130hd platform LEF/Lib/GDS files in `/OpenROAD-flow-scripts/flow/platforms/sky130hd/`.
- **Do not** hard-code absolute host paths in Tcl scripts; always use `/workspace/` (container path).
- **Do not** run `make` directly; always use the MCP flow tools.
- **Do not** skip `check_design` before synthesis.
